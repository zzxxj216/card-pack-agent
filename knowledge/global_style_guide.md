# Global Style Guide — 全局风格准则

所有 Agent 都会在生成前加载本文件。面向"一般性审美与叙事原则"，具体类目的差异化在 `categories/*.md`。

---

## 1. 视觉统一性

### 1.1 单包内必须统一
- 主色调：单包内主色最多 2 个色系（可选一个强调色）
- 字体：单包内标题字体不超过 2 种，正文字体统一
- 构图节奏：开头钩子卡（前 3-5）可以破格，其余保持视觉节奏

### 1.2 跨包的账号识别
- 字体家族：以 `serif`（叙事类）/ `sans-display`（情绪类）为主力
- Logo / 水印：右下角 8pt，不遮挡主体
- 卡贴尺寸：固定 9:16 (1080×1920)

---

## 2. 文字排版

### 2.1 字号档位（竖版 1080×1920 画布）
| 用途 | 字号 | 行距 |
|---|---|---|
| 超大钩子文字 | 80-120pt | 1.0 |
| 标题 | 48-72pt | 1.1 |
| 正文 | 28-40pt | 1.3 |
| 辅助信息 | 20-24pt | 1.4 |

### 2.2 排版铁律
- 单张卡贴主文字不超过 30 字
- 不压在人脸、手、视觉焦点上
- 文字与背景对比度 ≥ 4.5:1（WCAG AA）
- 避免"居中对齐 + 文字过长" → 视觉涣散

---

## 3. 叙事节奏原则

### 3.1 50 张卡贴包的黄金结构

```
[ 1-3  ]  钩子卡：抛问题/刺痛点/反常识（最强视觉冲击）
[ 4-8  ]  展开 1：铺设情绪/场景
[ 9-20 ]  展开 2：具体细节/证据/场景切换
[21-35 ]  高潮段：核心主张/最深情绪/最强画面
[36-45 ]  转场 / 反思 / 深化
[46-50 ]  收尾：升华 / CTA / 留白
```

张数不是死的。短包（20-30 张）、超长包（80+）按比例缩放。

### 3.2 留白原则
- 至少 20% 的卡贴给"留白时刻"：少字、慢节奏、视觉呼吸
- 不要 50 张都在输出观点，观众会疲劳

### 3.3 钩子强度
钩子卡必须满足**其中至少一个**：
- 反常识：颠覆惯常预期
- 情绪刺痛：戳中 universal regret / pain
- 视觉冲击：强烈的单一意象 + 大字
- 悬念制造："如果...""如果不是..."

---

## 4. 文案基调

### 4.1 语气
- 亲近但不谄媚：像朋友聊天，不装亲戚
- 克制但不冷漠：有情绪但不滥情
- 具体优于抽象：写"那碗凉掉的粥"不写"思念的味道"

### 4.2 人称
- 多用第二人称 "你" 而非 "我们"
- 避免频繁出现 "我"（除非叙事需要第一视角）

### 4.3 反复用的废话（避免）
- "在这个快节奏的时代..."
- "每个人心里都有..."
- "你有没有过这样的时刻..."（已饱和）
- "愿你..." 开头（blessing 类除外，已过度）

---

## 5. 情绪强度的"油腻线"

一句话：**比观众多一点情绪，不要比观众多太多**。

油腻信号（Evaluator 应 warn）：
- 连续多张出现 "破防" "泪目" "哭死" "狠狠共情"
- 自我代入过度："姐妹们我真的"
- 结尾强行升华："这就是人生啊"

## 6. CTA 使用

| 机制 | 推荐 CTA 强度 | 示例 |
|---|---|---|
| `resonance_healing` | `soft` | "你有类似时刻吗" |
| `regret_sting` | `none` / `soft` | 留白或 "评论区说说你的" |
| `contrast_twist` | `soft` | "最后一张给我震撼到了" |
| `blessing_ritual` | `soft` | "把这份祝福给到你想到的人" |
| `utility_share` | `hard` | "收藏备用 / 转发给他" |
| `aphorism_lesson` | `soft` | "认同请扣 1" |
| `conflict_tension` | `none` | 留白，让观众自发讨论 |

---

## 7. AI 生图的保底要求

- Prompt 必须包含风格 anchor（如 `film photography, 35mm, natural light`）
- 负面 prompt 必须包含文字类关键词（`text, watermark, logo, typography`）—— 卡贴上的文字用后期合成，不用 AI 生图
- 单包所有图用统一 seed 基或统一 style reference 图
- 人物类生成必须 post-check：手指数量、面部对称、眼神
