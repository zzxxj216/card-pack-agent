# Failure Library — 翻车案例库

每一条翻车都是真金白银换来的经验。本库比成功案例库更值得新成员先看。

**记录原则**：
- 每条必须有 **case_id**、**原因分析**、**规则抽取**
- 规则抽取要能直接写进 `categories/*.md` 或 `global_anti_patterns.md`
- 记录即使主观，也比遗忘好

---

## 记录模板

```markdown
### case-{id}: {简短标题}

- **日期**: YYYY-MM-DD
- **类目**: l1/l2
- **话题**: ...
- **表现**: tier (vs 同类目 baseline)
- **具体症状**: 用户差评的点 / 数据异常点
- **归因分析**:
  - 视觉层: ...
  - 文案层: ...
  - 叙事层: ...
  - 外部因素: ...
- **抽取规则**: 写成一句可被 prompt 约束的规则
- **规则落地**: 已写入 `xxx.md §xxx`
```
